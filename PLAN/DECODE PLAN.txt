🔐 DECODE: Execution Master Plan

Objective: Build a serverless, static PWA for QR Generation, text  Encryption and Decription platform.
Host: GitHub Pages (Zero Cost).

🛠️ PHASE 1: Setup & Configuration (The Skeleton)

[ ] 1.1 Initialize Project (Terminal)

npx create-next-app@latest decode --typescript --tailwind --eslint --no-src-dir --app --import-alias "@/*"
cd decode


[ ] 1.2 Install Core Libraries

# QR Code Engine (Handles Logos & Styling)
npm install qr-code-styling

# Icons & Animations
npm install lucide-react framer-motion

# Utilities
npm install clsx tailwind-merge


[ ] 1.3 Configure for GitHub Pages (next.config.js)
Action: Replace contents of next.config.js with:

/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',  // <--- Enables Static HTML Export
  images: {
    unoptimized: true, // <--- Required for GH Pages
  },
  basePath: '/decode', // <--- REPLACE 'decode' with repo name if not at root domain
};
module.exports = nextConfig;


📅 PHASE 2: Logic Engines (The Brains)

[ ] 2.1 QR Hook (hooks/useQRCode.ts)

Function: Wraps qr-code-styling to prevent "Window not found" errors in Next.js.

Inputs: url, color, logoUrl, shape ('square' | 'dots').

Action: Returns a ref to attach to a div and a download() function.

[ ] 2.2 Crypto Utilities (lib/crypto.ts)

Caesar Cipher:

export const caesarCipher = (str: string, shift: number): string => {
  return str.replace(/[a-zA-Z]/g, (char) => {
    const start = char <= 'Z' ? 65 : 97;
    return String.fromCharCode(((char.charCodeAt(0) - start + shift) % 26) + start);
  });
};


Base6orm:

export const toBase64 = (str: string) => typeof window !== 'undefined' ? window.btoa(str) : '';
export const fromBase64 = (str: string) => typeof window !== 'undefined' ? window.atob(str) : '';

[ ] 4.3 Review & Feedback (app/review/page.tsx)

Objective: Collect testimonials and screenshots sent directly to dedcoder@gmail.com.

Tech: Use EmailJS (Client-side email sending, free tier).

Form Fields:

Name & Email (<input>)

Review Message (<textarea>)

Screenshot (<input type="file"> - Note: EmailJS requires converting image to Base64 to attach, or uploading to cloud first. For MVP, stick to text or use a Base64 conversion utility).
